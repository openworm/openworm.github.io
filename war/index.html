Why?


If we ever hope to understand how the brain works and what happens when it suffers from disease, the most important thing we can do is to unleash our most powerful tools to organize the millions of facts that we know about the biology of the brain.

With the best scientists and engineers we can assemble the greatest puzzle of our time - what are all the events that happen inside a living brain?

Here at OpenWorm, we are starting this important journey with a practical step - by doing a full simulation of a very "simple" biological system, the nematode Caenorhabditis elegans.

We can’t do it alone. Please join us!

A talented team of researchers at the A.P. Ershov Institute of Informatics Systems SB RAS, Lab. of Complex Systems Simulation, consisting of Andrey Palyanov, Sergey Khayrulin, and Alexander Dibert developed an initial neuromechanical model of the c. elegans in 3D. Source code for this is available.

The original Cyber Elegans team joined the Open Worm project to incorporate greater biological realism into their model, simulating the soft body of the C. elegans rather than using a rigid body model, and to break-down their code base into a modular and generic simulation engine.

This video shows a neuronal simulation demo built on the current version of the OpenWorm simulation engine. In the video one of the 302 Hodgkin-Huxley neurons (simulated in parallel on GPU native kernel via OpenCL) is displayed and by changing the external current applied the spike shape and firing rate are updated.

This rather basic setup is just meant to be a proof of concept of the rather complex architectural stack involved behind the scenes (see SimulationFrameworkConcepts for more details).

We have implemented an algorithm called Smoothed Particle Hydrodynamics (SPH) (see reference for more) to simulate the body of the worm and its environment.

We have not yet incorporated the worm anatomy into this solver, although this will be the next step. If you are running Windows, you can download a current version of our alpha implementation of PCISPH as a binary here. The source code for this demo is available online here. The SPH algorithm has also been implemented as a solver for the OpenWorm simulation engine.

We have converted all 302 neurons described in the WormBase Virtual Worm Blender files into multi-compartmental neuronal models. To make the models open and accessible, we have built the neurons in a neuroConstruct project, which outputs NeuroML. The latest version of this project is freely available online.

This captures the positions of the cells in context with each other and gives us a place to start building descriptions of the synaptic junctions and the ion channels for each cell.

A major component of our strategy with this project is to use optimization techniques like genetic algorithms to help fill gaps in our knowledge. During release 2, our goal is to nail down the details for body wall muscle cell MDL08.

We are building on the work of Boyle & Cohen 2007, who built a model of a C. elegans body wall muscle cell. As part of their work, they used a genetic algorithm to tune the exact values of key parameters used. Using a generous donation of the code used in this publication, and extending it, we have reproduced the results of Boyle and Cohen. This document summarizes out initial effort.

The OpenWorm Browser enables ready access to a cell-by-cell 3D representation of the nematode C. elegans in a WebGL enabled browser.

The C. elegans 3D model is provided by Christian Grove at the VirtualWorm project (WormBase, CalTech).

Our browser is based on WebGL technology and built on top of the open-3d-viewer (ex-"Google Body Browser").

As any other software developed as part of the OpenWorm project, the OpenWorm browser is completely open sourced.

OpenWorm technologies


Java is the core of our flexible simulation engine platform.

We are using OpenCL via JavaCL bindings to leverage GPUs in our simulations.

Python enables us to quickly hack together powerful scripts for many different purposes.

We distribute our systems in the “cloud” of Amazon Web Services to enable our international contributors access to computational resources wherever they are.

We use NeuroML to build the electrical / chemical models of neurons in our simulations.

We use neuroConstruct paired up with NeuroML in our simulations.

Our website is developed and hosted on the Google AppEngine platform.
